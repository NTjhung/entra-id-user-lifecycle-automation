# Offboarding Checklist

## Purpose

This checklist documents the steps an IAM team should follow when a user leaves the organization. The goal is to remove access quickly, reduce security risk, and keep evidence for audit purposes.

## Offboarding Steps

1. Confirm the user's termination date with HR or the manager.
2. Disable the user's Microsoft Entra ID account.
3. Revoke active user sessions.
4. Reset the user's password.
5. Remove the user from assigned security groups.
6. Remove access to sensitive applications.
7. Remove privileged roles if assigned.
8. Confirm mailbox, OneDrive, or shared data ownership needs.
9. Document all completed actions in the ticket.
10. Export or save evidence for audit review.

## Access Removal Checklist

| Task | Status |
|---|---|
| User account disabled | Not Started |
| Sessions revoked | Not Started |
| Password reset | Not Started |
| Security groups removed | Not Started |
| Application access removed | Not Started |
| Privileged roles removed | Not Started |
| Evidence saved | Not Started |
| Ticket updated | Not Started |

## Notes

Offboarding should be completed as soon as possible after termination is confirmed. Delayed access removal can create security, compliance, and audit risks.
